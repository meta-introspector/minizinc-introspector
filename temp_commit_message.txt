refactor: Parameterize shell scripts with .env file

This commit refactors the build and run shell scripts to use environment variables defined in a new `.env` file, replacing hardcoded paths.

- **`.env`**: A new file introduced in the project root to centralize path configurations.
- **`scripts/build_gecode.sh`**: Updated to source `.env` and use `GECODE_BUILD_DIR`.
- **`scripts/build_libminizinc.sh`**: Updated to source `.env` and use `LIBMINIZINC_BUILD_DIR`, `MINIZINC_PROJECT_ROOT`, and `GECODE_BUILD_DIR`.
- **`scripts/setup_minizinc_solvers.sh`**: Updated to source `.env` and use `MINIZINC_USER_SOLVERS_DIR` and `GECODE_BUILD_DIR`.
- **`scripts/run_embedding_model.sh`**: Updated to source `.env` and use `LIBMINIZINC_BUILD_DIR`, `MINIZINC_MODELS_DIR`, and `MINIZINC_DATA_DIR`.

This improves the portability, maintainability, and reusability of the project's automation scripts.
